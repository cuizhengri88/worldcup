# 동적 월드컵 토너먼트 대진표 기획서

## 목표
현재 `2026.html`에 직접 작성된 팀, 확률, 분석 문구, 승패 상태를 JSON 데이터 기반 구조로 분리한다. 사용자가 32강부터 실제 경기 결과에 따라 승리 팀을 선택하면, 선택된 팀 정보와 각 팀의 전력 JSON 데이터를 바탕으로 다음 라운드 대진표가 자동으로 생성되도록 만든다.

최종 목표는 정적인 예측표가 아니라, 실제 결과를 입력할 때마다 16강, 8강, 4강, 결승 대진이 동적으로 바뀌는 토너먼트 시뮬레이터다.

## 핵심 사용자 흐름
1. 사용자는 소조 경기 결과 영역에서 32강에 올라갈 팀 후보를 확인한다.
2. 32강 대진표의 각 슬롯은 selectbox 형태로 표시된다.
3. 32강 selectbox에서는 소조 결과에 있는 진출 팀 목록 중 팀을 선택한다.
4. 한 경기의 두 팀이 모두 선택되면, JSON에 저장된 전력 수치를 비교해 승률과 분석 문구를 생성한다.
5. 사용자는 해당 경기의 승리 팀을 selectbox 또는 승자 선택 컨트롤로 지정한다.
6. 선택된 승리 팀은 다음 라운드의 후보 목록에 자동으로 포함된다.
7. 16강 selectbox에서는 32강에서 승자로 선택된 팀만 선택할 수 있다.
8. 8강 selectbox에서는 16강에서 승자로 선택된 팀만 선택할 수 있다.
9. 4강과 결승도 같은 방식으로 이전 라운드 승자만 선택 가능하다.
10. 모든 라운드 결과가 입력되면 최종 우승 팀과 전체 토너먼트 결과가 완성된다.

## 데이터 분리 방향
기존 HTML에 들어 있는 정보는 크게 세 종류로 분리한다.

- 팀 전력 데이터: 팀별 공격력, 수비력, 중원, 경험, 최근 폼, 주요 선수, 설명 문구
- 소조 진출 데이터: 각 조별 1위, 2위, 와일드카드 등 32강 후보 팀 목록
- 대진 데이터: 32강 초기 매치업, 라운드 구조, 다음 라운드 연결 정보
- 결과 데이터: 사용자가 선택한 실제 승자, 경기 스코어, 선택 시각, 확정 여부

## JSON 설계

### `data/teams.json`
팀별 고정 전력 데이터를 저장한다. 이 데이터는 경기마다 승률을 계산하는 기준값이 된다.

```json
{
  "KOR": {
    "name": "대한민국",
    "flag": "🇰🇷",
    "group": "A",
    "seed": 2,
    "rating": {
      "overall": 82,
      "attack": 84,
      "midfield": 81,
      "defense": 78,
      "depth": 76,
      "experience": 80,
      "form": 83
    },
    "players": ["손흥민", "이강인", "김민재"],
    "style": "빠른 전환과 측면 침투",
    "summary": "공격 전환 속도와 핵심 선수의 결정력이 강점이다."
  }
}
```

### `data/group-qualifiers.json`
소조 결과를 바탕으로 32강 selectbox에서 선택 가능한 팀 후보를 저장한다. 32강 대진 슬롯은 이 파일에 있는 팀만 선택할 수 있다.

```json
{
  "groups": [
    {
      "id": "A",
      "label": "A조",
      "qualifiers": [
        { "teamId": "MEX", "rank": 1 },
        { "teamId": "KOR", "rank": 2 },
        { "teamId": "CZE", "rank": "wildcard" }
      ]
    }
  ],
  "allQualifiedTeamIds": ["MEX", "KOR", "CZE"]
}
```

32강 selectbox는 기본적으로 `allQualifiedTeamIds`를 사용한다. 필요하면 경기별 선택 후보를 제한할 수도 있다.

### `data/bracket.json`
초기 32강 대진과 다음 라운드 연결 규칙을 저장한다.

```json
{
  "rounds": [
    {
      "id": "round32",
      "label": "32강전",
      "matches": [
        {
          "id": "M01",
          "slot": 1,
          "teamA": null,
          "teamB": null,
          "teamSource": "groupQualifiers",
          "nextMatchId": "R16-A",
          "nextSlot": "teamA"
        }
      ]
    },
    {
      "id": "round16",
      "label": "16강전",
      "matches": [
        {
          "id": "R16-A",
          "slot": 1,
          "teamA": null,
          "teamB": null,
          "teamSource": "round32Winners",
          "nextMatchId": "QF-1",
          "nextSlot": "teamA"
        }
      ]
    }
  ]
}
```

### `data/results.json`
사용자가 입력한 실제 경기 결과를 저장한다. 처음에는 비어 있어도 된다.

```json
{
  "matches": {
    "M01": {
      "teamA": "MEX",
      "teamB": "CIV",
      "winner": "MEX",
      "score": "2-1",
      "confirmed": true
    }
  }
}
```

브라우저만으로 동작시키는 1차 버전에서는 파일에 직접 저장하지 않고 `localStorage`에 저장한다. 이후 서버나 백엔드를 붙이면 `results.json` 또는 API로 영구 저장할 수 있다.

## 전력 비교 계산 방식
1차 버전은 단순하고 설명 가능한 가중치 계산을 사용한다.

```text
teamScore =
  overall * 0.30 +
  attack * 0.18 +
  midfield * 0.16 +
  defense * 0.16 +
  depth * 0.08 +
  experience * 0.06 +
  form * 0.06
```

두 팀의 점수를 비교해 기본 승률을 만든다.

```text
teamAWinRate = teamAScore / (teamAScore + teamBScore)
teamBWinRate = 1 - teamAWinRate
```

표시할 때는 너무 극단적인 수치가 나오지 않도록 35%~65% 범위로 보정한다. 예외적으로 전력 차이가 매우 큰 경우에는 70%까지 허용할 수 있다.

## 동적 렌더링 규칙
기존 HTML의 `.bracket-column`, `.matchup-box`, `.team-row-btn` 구조는 최대한 유지한다. 다만 HTML에 경기 카드를 직접 박아 넣지 않고 JavaScript가 JSON을 읽어 생성한다.

렌더링 순서:
1. `teams.json` 로드
2. `bracket.json` 로드
3. `localStorage` 또는 `results.json`의 결과 로드
4. 결과를 bracket 상태에 반영
5. 각 라운드별 컬럼 생성
6. 각 경기별 카드 생성
7. 각 경기 슬롯에 팀 선택 selectbox 생성
8. 32강 selectbox는 소조 진출 팀 목록을 후보로 사용
9. 16강 이후 selectbox는 직전 라운드 승자 목록을 후보로 사용
10. 팀이 둘 다 선택된 경기는 승률과 분석 문구 표시
11. 아직 선택 가능한 후보가 없는 슬롯은 `이전 라운드 결과 대기` 상태로 표시

## Selectbox 기반 대진 입력 UX
각 경기 카드는 두 개의 팀 선택 selectbox와 하나의 승자 선택 selectbox를 가진다.

```text
MATCH 01
[팀 A 선택 ▼] vs [팀 B 선택 ▼]
예상 승률: 팀 A 58% / 팀 B 42%
승자: [승리 팀 선택 ▼]
```

32강 팀 선택 규칙:
- 후보 목록은 `group-qualifiers.json`의 소조 진출 팀 전체 목록에서 가져온다.
- 이미 같은 32강 경기의 반대 슬롯에서 선택된 팀은 선택 불가 처리한다.
- 같은 라운드의 다른 경기에서 이미 선택된 팀도 중복 선택 불가 처리하는 것을 기본 정책으로 한다.
- 팀 A와 팀 B가 모두 선택되면 두 팀의 JSON 전력 수치로 승률을 계산한다.

16강 이후 팀 선택 규칙:
- 후보 목록은 이전 라운드에서 승자로 선택된 팀만 사용한다.
- 예를 들어 16강은 32강 승자 목록, 8강은 16강 승자 목록에서만 선택 가능하다.
- 해당 라운드의 후보가 아직 충분하지 않으면 selectbox는 비활성화하고 `이전 라운드 승자 대기` 문구를 보여준다.
- 사용자가 이전 라운드 승자를 바꾸면 이후 라운드의 선택값과 결과는 자동 초기화한다.

승자 선택 규칙:
- 승자 selectbox는 해당 경기의 팀 A와 팀 B가 모두 선택된 뒤 활성화된다.
- 승자 후보는 해당 경기의 팀 A, 팀 B 두 팀으로 제한한다.
- 승자를 선택하면 다음 라운드 후보 목록에 반영된다.
- 승자를 변경하면 그 팀이 영향을 준 이후 라운드 선택값은 초기화된다.

## 실제 결과 선택 UX
각 경기 카드에서 selectbox로 팀과 승자를 선택한다. 기존처럼 팀 행을 클릭해 승자를 고르는 방식은 보조 기능으로만 둔다.

- 팀 선택 전: selectbox가 `팀 선택` placeholder를 표시
- 팀 선택 후: 선택된 팀 정보, 국기, 전력 요약 표시
- 승자 선택 전: 승자 selectbox 비활성화
- 승자 선택 후: 승자는 `winner`, 패자는 `loser` 스타일 적용
- 다음 라운드 슬롯 자동 업데이트
- 이미 선택된 경기의 승자를 바꾸면 이후 라운드 결과 중 영향을 받는 경기들을 초기화
- 선택 확정 기능은 2차 개발에서 추가 가능

## 다음 라운드 자동 생성 규칙
경기에는 `nextMatchId`와 `nextSlot`을 둔다. 예를 들어 `M01`의 승자가 `R16-A.teamA`로 이동한다.

승자 선택 시 처리:
1. 현재 경기 결과 저장
2. 현재 라운드의 승자 목록 갱신
3. 다음 라운드 selectbox 후보 목록 갱신
4. 기존 방식처럼 특정 슬롯 자동 배치가 필요한 경우 `nextMatchId`, `nextSlot`에 따라 기본값 지정
5. 다음 경기의 두 슬롯이 모두 채워졌는지 확인
6. 채워졌다면 전력 비교 수치 계산
7. 화면 전체 또는 해당 라운드 이후만 다시 렌더링

선택형 대진표에서는 다음 라운드 슬롯을 무조건 자동 배치하는 방식과, 후보 목록만 제공하고 사용자가 직접 selectbox에서 고르는 방식이 모두 가능하다. 1차 버전은 사용자가 직접 선택할 수 있게 하되, 이전 라운드 승자만 후보로 제한한다.

## 분석 문구 생성 방식
1차 버전은 템플릿 기반으로 만든다.

예시:
```text
{teamA}는 공격력과 최근 폼에서 우위를 보이며, {teamB}는 수비 조직력에서 강점이 있습니다.
종합 전력 수치 기준 {winner}의 우세가 예상되지만, 토너먼트 특성상 경기 흐름에 따라 변수가 발생할 수 있습니다.
```

2차 버전에서는 팀 스타일, 주요 선수, 강점 차이를 조합해 더 자연스러운 문장을 만든다.

- 모달 표시

## UI 변경 방향
현재 어두운 테마와 카드 스타일은 유지한다. 정적인 카드에서 동적 카드로 바뀌더라도 사용자는 기존 화면과 비슷하게 느껴야 한다.

추가할 UI:
- 상단에 `결과 초기화` 버튼
- 각 경기 카드에 팀 선택 selectbox 2개
- 각 경기 카드에 승자 선택 selectbox 1개
- 각 selectbox에 `팀 선택`, `승자 선택` placeholder 표시
- 선택된 팀의 전력 요약 표시
- 아직 선택 후보가 없는 경기에는 `이전 라운드 승자 대기` 표시
- 실제 결과가 입력된 라운드는 헤더에 완료 수 표시

예시:
```text
32강전 12/16 완료
16강전 4/8 완료
8강전 대기 중
```

## 개발 순서
1. 현재 HTML의 팀/경기 데이터를 표로 정리한다.
2. 팀 식별자 규칙을 정한다. 예: 대한민국 `KOR`, 프랑스 `FRA`
3. `data/teams.json`을 만든다.
4. `data/group-qualifiers.json`을 만든다.
5. `data/bracket.json`을 만든다.
6. 기존 HTML의 브래킷 내부를 빈 컨테이너로 바꾼다.
7. JS에서 JSON을 로드해 브래킷을 렌더링한다.
8. 32강 팀 선택 selectbox를 붙인다.
9. 경기별 승자 선택 selectbox를 붙인다.
10. 이전 라운드 승자만 다음 라운드 후보가 되도록 연결한다.
11. 전력 수치 기반 승률 계산을 붙인다.
12. localStorage 저장/복원 기능을 붙인다.
13. 모바일과 데스크톱 레이아웃을 검증한다.

## 주의할 점
- 팀 이름 문자열을 직접 비교하지 말고 항상 팀 ID를 기준으로 처리한다.
- selectbox option의 value도 팀 이름이 아니라 팀 ID를 사용한다.
- 32강 후보 목록은 소조 진출 데이터에서 가져오고, 16강 이후 후보 목록은 이전 라운드 결과에서 계산한다.
- 같은 라운드에서 동일 팀이 여러 경기 슬롯에 중복 선택되지 않도록 막는다.
- 승자를 변경하면 그 팀이 올라가서 만든 이후 라운드 결과를 무효화해야 한다.
- JSON 로드 방식 때문에 로컬 파일을 브라우저에서 직접 열면 CORS 문제가 생길 수 있다. 개발 중에는 간단한 로컬 서버를 사용하는 것이 좋다.
- 기존 HTML은 한글 인코딩이 깨져 보이는 부분이 있으므로, 리팩터링 시 파일 전체를 UTF-8로 정리하는 작업도 같이 고려한다.

## 완료 기준
- HTML에 직접 박힌 경기 카드 없이 JSON만 수정해 32강 대진을 바꿀 수 있다.
- 32강 selectbox에서 소조 진출 팀을 선택할 수 있다.
- 16강 selectbox에서는 32강 승자로 선택된 팀만 선택할 수 있다.
- 이후 라운드도 직전 라운드 승자만 후보로 표시된다.
- 사용자가 승자를 선택하면 다음 라운드 후보 목록이 자동 갱신된다.
- 각 경기의 승률과 분석 문구가 팀 전력 JSON을 기반으로 표시된다.
- 새로고침 후에도 입력한 결과가 유지된다.
- 결과 초기화 버튼으로 처음 상태로 돌아갈 수 있다.
